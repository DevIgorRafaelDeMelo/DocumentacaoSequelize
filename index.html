Descrição do projeto

Esta documentação fornece uma visão geral do curso avançado em ORM, NodeJs e MySql, incluindo sua relação com a missão, objetivos de aprendizado e iniciativa, contexto para a próxima geração e lições aprendidas. O curso ensina como utilizar o Sequelize com Javascript, utilizando a engine do NodeJs para criar um projeto back-end. O foco principal é a manipulação de dados do MySql, utilizando o protocolo do Sequelize. A ideia é criar um back-end para uma escola, onde serão feitos cadastros de estudantes, matrículas, turmas e níveis de graduação, criando as conexões de chaves e definindo os modelos de dados. Também foram utilizados seeds para popular o banco com dados iniciais. Além disso, foram implementadas diversas funcionalidades para manipulação dos dados no banco utilizando sempre o Sequelize.

Durante o projeto, foram utilizadas as seguintes ferramentas:

  • ORM (Object-Relational Mapping) - NodeJs
  • JavaScript
  • Sequelize
  • MySQL
  • JSON
  • Postman
  • Api.rest
  • Npm

Estrutura do Projeto

Instalando projeto

Iniciando com o comando no terminal.

— npm i

Vão ser estalados todas as dependências do projeto utilizadas.

  • Sequelize-cli
  • body-parser
  • express
  • mysql2
  • path
  • nodemon

Iniciando o projeto

Insira o comando no terminal.

— npm run start

O servidor local ira iniciar

Solicitações seguindo o protocolo Rest

Parâmetros e definição de retorno:

GET / http://localhost/pessoas

Retornara um JSON de todos os registros de pessoas cadastrada no banco, com os seguintes valores.

GET / http://localhost/pessoas/scopes

Retorna todos os registros de pessoas ativas cadastrada no banco.

GET / http://localhost/pessoas/:id

Passando o parâmetro ID ele solicitara o elemento pelo seu id

POST / http://localhost/pessoas

Enviando um POST com parâmetros no corpo da solicitação enviando os seguintes dados no formato JSON, isso criara um novo cadastro pessoa.

PUT / http://localhost/pessoas/:id

Enviando um PUT com parâmetros no corpo da solicitação enviando os seguintes dados no formato JSON, isso Atualizara os dados cadastrados no id determinado.

DELETE / http://localhost/pessoas/:id

Solicitando DELETE com o parâmetro de id os registro ficarão oculto para próximas solicitações, para melhor manutenção dos registro o sequelize cria uma nova tabela determinando a data da exclusão, esses continuarão no banco porém ocultos, apenas com uma nova requisição seguida abaixo pode ser feito essa reversão restaurando o elemento.

POST / http://localhost/pessoas/:id/restaura

Passando o ID determinado ele retorna registros já “excluídos” deixando eles acessíveis.

GET / http://localhost/pessoas/:id/matricula/:idM

Esse parâmetro retornara id e matricula determinado.

GET / http://localhost/pessoas/:id/matricula

Esse parâmetro retornara todas as matriculas da pessoas determinada .

GET / http://localhost/pessoas/matricula/:id/confirmadas

Esse parâmetro retornara a turma determinada pelo ID trazendo todos as Matriculas cadastradas na turma.

POST / http://localhost/pessoas/:id/matricula

Enviando um POST com parâmetros no corpo da solicitação enviando os seguintes dados no formato JSON, isso criara uma nova Matricula.

PUT / http://localhost/pessoas/:id/matricula/:idM

Enviando um PUT com parâmetros de ID e IDM atualizara os dados da matricula do id especifico.

DELETE / http://localhost/pessoas/:id/matricula/:idM

Solicitando DELETE com o parâmetro de id E IDm os registro ficarão oculto para próximas solicitações, para melhor manutenção dos registro o sequelize cria uma nova tabela determinando a data da exclusão, esses continuarão no banco porém ocultos, apenas com uma nova requisição seguida abaixo pode ser feito essa reversão restaurando o elemento.

POST / http://localhost/:id/restaura

Passando o ID determinado ele retorna registros já “excluídos” deixando eles acessíveis.
